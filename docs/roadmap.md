# ロードマップ

## Phase 0: 企画と設計

- プロダクト概要を決める
- MVPの範囲を決める
- 技術方針を決める
- 教材データの形式を決める
- アプリ名とデザイン方針を決める
- 基礎から応用までの学習カリキュラムを決める

## Phase 1: 最小プロトタイプ

- アプリの画面構成を作る
- ホーム画面を作る
- レッスン一覧を作る
- タイ文字カード画面を作る
- フラッシュカード画面を作る
- サンプルの文字教材を数件入れる
- 独自ローマ字表記を表示する
- 中子音・高子音・低子音の分類を表示する
- スマホ幅で主要画面が使えるようにする

## Phase 2: クイズと進捗

- 4択クイズを作る
- 正解・不正解の表示を作る
- タイ文字から独自ローマ字表記を選ぶ問題を作る
- 独自ローマ字表記からタイ文字を選ぶ問題を作る
- タイ文字から子音分類を選ぶ問題を作る
- localStorageで進捗を保存する
- 間違えた問題を復習できるようにする

## Phase 3: タイ語教材を増やす

- 中子音を増やす
- 高子音を増やす
- 低子音を増やす
- 母音記号を増やす
- 声調の入門レッスンを作る
- 基本単語を増やす

## Phase 4: 学習体験を整える

- スマホ画面を調整する
- タブレットとPC画面を調整する
- 学習完了画面を作る
- 進捗表示を見やすくする
- 復習導線を改善する

## Phase 5: 拡張検討

- PWA対応
- 音声再生
- 発音録音
- AI会話練習
- ユーザーアカウント
- 進捗同期
- スマホアプリ化

## 直近の次アクション

1. 最初に入れる中子音・高子音・低子音の範囲を決める
2. 技術スタックを確定する
3. モバイルファーストで画面構成をラフに決める
4. プロトタイプ実装を始める
