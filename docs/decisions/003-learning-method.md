# 003: 学習方法は体系型、独自ローマ字表記中心、音声なしMVPにする

## 状態

採用

## 決定

`タイ文字ノート` は、基礎から応用まで体系的に学べる構成にする。

読み方は基本的にカタカナを使わず、独自ローマ字表記を使う。

MVPでは音声なしで作る。

子音は一度に全部出さず、中子音・高子音・低子音に分類して、少しずつ増やす。

## 理由

タイ語は、文字・発音・声調・子音分類の関係を理解しないと、単語や会話の学習でつまずきやすい。

カタカナ読みは入り口としてわかりやすい一方で、タイ語の音や声調を正確に扱いにくい。

そのため、最初から独自ローマ字表記に慣れる設計にする。

音声は重要だが、MVPでは画面構成、教材データ、分類学習、クイズ体験を先に固める。

## 影響

- 文字カードには独自ローマ字表記と子音分類を表示する
- クイズには子音分類問題を入れる
- UIには原則としてカタカナ読みを出さない
- 音声再生は後続フェーズに回す
- 教材データには将来の音声追加に備えて `audioPath` を残す
