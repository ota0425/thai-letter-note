# 002: アプリ名、学習軸、デザイン方針を決める

## 状態

採用

## 決定

アプリ名は `タイ文字ノート` を仮採用する。

初期の学習軸は、タイ文字を覚える基礎学習にする。

Webアプリとして開発を始めるが、最初からモバイル対応を意識したレスポンシブUIにする。

デザインは、落ち着いた学習ツールの雰囲気にする。

## 理由

タイ語初心者にとって、文字・発音・声調は大きなつまずきになりやすい。

会話フレーズや単語暗記だけで始めるよりも、タイ文字の形と音を結びつける学習体験を先に整える。

`タイ文字ノート` という名前は、学習対象がわかりやすく、落ち着いて基礎を積み上げる印象に合っている。

## モバイル対応について

Webアプリでも、スマホやタブレットに対応できる。

最初から以下を前提にする。

- モバイルファーストで画面を設計する
- レスポンシブCSSでスマホ、タブレット、PCに対応する
- タップしやすい操作部品にする
- 後からPWA化できる構成にする
- 必要になればReact NativeやCapacitorでスマホアプリ化を検討する

## 影響

- MVPではタイ文字カードと文字クイズを優先する
- 会話レッスンは後続フェーズに回す
- 画面設計ではスマホ幅を最初に確認する
- デザインでは装飾より読みやすさを優先する
